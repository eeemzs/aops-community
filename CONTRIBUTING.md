# Contributing to AOPS Community

Thank you for helping improve AOPS Community.

## Source contribution workflow

1. Fork and clone eeemzs/aops-community.
2. Use Node.js 22.9.0 or newer and pnpm@11.9.0.
3. Install the frozen workspace with pnpm install --frozen-lockfile.
4. Make a focused change and add or update the relevant tests.
5. Run pnpm verify before opening a pull request.

Keep credentials, local environment files, generated database data, build output, and package-manager caches out of commits. Security-sensitive findings should follow [SECURITY.md](SECURITY.md) instead of being disclosed in a public issue.

By contributing, you agree to the [Developer Certificate of Origin](DCO.md). Project maintainers may decline, revise, or remove contributions as part of maintaining the project and may require additional validation even when automated checks pass. Only maintainers merge pull requests.
