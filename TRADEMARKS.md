# AOPS Project Marks

The Apache License 2.0 grants copyright and patent permissions, but it does not
grant trademark rights except for reasonable use in describing the origin of
the software and reproducing required notices.

This policy applies to the names **AOPS**, **AOPS Community**, and official AOPS
logos or visual identifiers (the "project marks"). It does not state that any
mark is registered.

## Uses that do not require separate permission

You may:

- truthfully state that software is based on, forked from, or compatible with
  AOPS Community;
- link to the official repository and reproduce attribution required by the
  Apache License 2.0;
- discuss, review, teach, or write about the project;
- redistribute an unmodified official release while accurately identifying its
  version and source.

These uses must not suggest sponsorship, certification, endorsement, or an
official relationship that does not exist.

## Modified distributions and forks

A materially modified distribution must use a distinct product name and visual
identity. It may say, for example, `based on AOPS Community`, but it must not use
`AOPS` or `AOPS Community` as its primary product, organization, package,
service, domain, or application name. Preserve legal attribution while making
the fork relationship and responsible maintainer clear.

Do not use the project marks in a way that is deceptive, confusing, unlawful,
or damaging to the project or its users. Do not register confusingly similar
names, domains, accounts, or logos.

## No effect on the open-source license

This policy does not limit rights granted by the Apache License 2.0 in the source
code, and it does not add a field-of-use restriction. It governs only use of the
project marks. To request permission for a use not covered here, contact the
repository owner `@eeemzs` before launch and describe the proposed name, design,
context, and duration.
