Compose-first quickstart

Run it without a cloud account.

The generated candidate pins its base images and pnpm version. The initializer writes independent create-once PostgreSQL and ChatV3 secrets to a local, ignored file.

trusted local shell
$ node deploy/community/init-env.mjs
$ docker compose up --build -d --wait
$ docker compose exec app \
    node deploy/community/demo.mjs seed

Open  http://127.0.0.1:5900

$ docker compose exec app \
    node deploy/community/chatv3-smoke.mjs
$ docker compose exec app \
    node deploy/community/demo.mjs reset

Responsible by construction

Honest boundaries are product features.

The Community profile is an allowlist. Excluded domains are absent from routes, tools, CLI commands, navigation, source closure, and runtime packages—not merely hidden with CSS.

What this candidate proves

  • single-operator, self-hosted workflow on a trusted machine
  • loopback-only host edge and private PostgreSQL
  • project-scoped Docman, Projectman, and Agentspace records
  • server-encrypted local ChatV3 flow with secret-safe invite handling
  • deterministic source and evidence trees

What it does not claim

  • Internet-facing or multi-tenant security
  • interactive AuthV2/RBAC, billing, or project sharing
  • Fileman or Tasker/Runner product surfaces
  • end-to-end encryption for every chat mode
  • hosted cloud recovery, abuse controls, or public-release readiness
Publication is intentionally blocked. The source candidate is licensed under Apache-2.0 and carries reviewed LICENSE and NOTICE files. Public release still requires SBOM/provenance, migration safety, independent release validation, and explicit operator authorization. No public publish is authorized.
